## Migration Tool

The `dinv migrate` command (v3.0070) imports data from old aard_inventory state files into dinv's SQLite database. It migrates items, priorities, equipment sets, consumables, stat bonuses, and configuration settings. Old state files are never modified.

### Technical approach

Old aard_inventory state files are serialized Lua table assignments (e.g., `inv.items.table[id].stats.level = 41`). The migration tool loads these files using Lua's `loadstring()` rather than a custom line-by-line parser. This approach was chosen because:

1. **Reliability:** The files are valid Lua by design — they were generated by the plugin's own serialization code. Using `loadstring` leverages Lua's own parser, which handles all edge cases (quoted strings with special characters, color codes, nested tables, numeric precision) without requiring us to re-implement a Lua parser.

2. **Safety:** The files are locally generated by the user's own plugin installation. They contain only table assignment statements. The code is executed in a sandboxed environment with `setfenv` to prevent side effects.

3. **Simplicity:** A line-by-line regex parser would need to handle indentation-based nesting, multiple quoting styles, type detection (string vs number vs boolean vs table), and escape sequences. This is essentially writing a subset of the Lua parser — fragile and unnecessary when the real parser is available.

### Validation

After migrating each data type, row counts are validated against the number of entries parsed from the old state files. If any count mismatches, the entire transaction is rolled back and the pre-migration backup remains available for restoration.
